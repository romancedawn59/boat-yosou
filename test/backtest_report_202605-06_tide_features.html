検証⑩ 潮汐特徴量の有効性(2026年5〜6月・検証のみ)

実施: 2026-07-12 / 学習: 2026-05-01より前の255,997行 / 評価: 2026-05-01〜06-30の対象5場 1,883レース(検証⑦⑧⑨と同じ枠組み)。 本検証は最初から「検証のみ」であり、結果が良くても本番導入はしない(2026-09-01のv2判断材料)。

判定: 効果なし(ノイズ)。 事前登録した予測「本物なら潮汐場(江戸川・平和島・若松)でのみ効き、対照場(常滑・尼崎)では効かない」に対し、 実際は潮汐場でも改善が出なかった(江戸川-0.0021 / 平和島-0.0010 / 若松+0.0002のAUC差)。 本命勝負所の回収率も152.6%→148.6%と微悪化。v2判断(2026-09-01)では「潮汐は現行特徴量の追加としては不要」が結論。

事前登録した反証可能な予測(実行前に固定)

  • 場マスタ調査: 江戸川=汽水・河川 / 平和島=海水・運河 / 若松=海水・洞海湾直結 → 干満の影響あり(潮汐場)
  • 常滑=水門で干満皆無 / 尼崎=淡水プール → 影響なし(対照場)
  • 予測: 潮汐特徴量が本物なら、効果は潮汐3場でのみ現れ、対照2場では現れない。 5場一様に"効いた/悪化した"場合はノイズと判定する

データと特徴量

  • 潮位データ: 気象庁「潮位表」(天文潮汐の推算値・決定論的)。2025〜2026年の4地点 (東京TK=江戸川・平和島 / 名古屋NG=常滑 / 大阪OS=尼崎 / 門司MO=若松)を test/fetch_tide.py で一括取得し、tideテーブルに1時間刻みで70,080件格納。評価行の充足率100%
  • tide_level: 締切時刻の潮位(cm) / tide_delta: 1時間の潮位変化(正=上げ潮) / tide_range_day: その日の潮位差(大潮・小潮の代理変数)
  • 潮汐は事前に確定している推算値のためリークなし。対象5場以外の場はNaN(LightGBMの欠損分岐に委ねる)

v2判断(2026-09-01)への引き継ぎ

  • 結論: 潮汐(潮位・干満・大潮小潮)は現行の特徴量セットへの追加としては効果なし
  • ただし本検証が否定したのは「締切時刻の潮位を特徴量に足す」形のみ。 風×潮の交互作用や、江戸川の「うねり」発生条件のような局所仮説までは棄却していない
  • 潮位データ(tideテーブル・70,080件)と取得の仕組み(test/fetch_tide.py)は資産として残る。 再検証は fetch_tide.py → verify_tide_features.py の2コマンド
  • 本番(features.py・モデル・買い目構成)には一切手を加えていない(v1凍結中)

⚠ 単一期間(5〜6月)の検証。ただし対照群つきの事前登録予測を先に固定してから 実行しており、「潮汐場でのみ改善」という支持パターンは観測されなかった。
