検証⑨ 進入コース特徴量(イン屋/アウト屋傾向)の有効性(2026年5〜6月)

実施: 2026-07-12 / 学習: 2026-05-01より前の255,997行 / 評価: 2026-05-01〜06-30の対象5場 1,883レース(検証⑦⑧と同じ枠組み)

判定: 不採用(経過観察)。 モデル精度は横ばい(AUC 0.8490→0.8488)なのに、実際に買う本命勝負所の回収率が 152.6%→142.8%(-9.8pt)、最大1発を除くと98.2%→88.7%と明確に悪化した。 データ(results.course)は日次収集で自動蓄積が続くため、数ヶ月後にデータが増えた時点で py -X utf8 test/verify_course_features.py の再実行だけで再検証できる。

背景とデータ状況

  • 現行モデルは枠番(lane)しか見ておらず、選手のコース取り傾向(イン屋/アウト屋)は特徴量にない
  • results.course(実際の進入コース)は収集済み: 325,173件・充足率98.1%・期間2025-07-15〜2026-07-11。 枠番≠コース(前づけ等)は全スタートの9.4%
  • コース別1着率(学習期間): 1コース55.8% / 2コース13.2% / 3コース13.1% / 4コース10.7% / 5コース6.4% / 6コース2.0%
  • 当日の進入コースはレース前に分からないため、使えるのは「選手の過去のコース取り傾向」のみ (features.pyの方針「results由来の当日情報は使わない」に抵触しない設計)

経過観察の運用(今回の成果物)

  • データ取得・保存: 追加作業なし。results.courseは日次収集(collect.py)で今後も自動蓄積される
  • 分析の仕組み: test/verify_course_features.py に保存(特徴量算出+2モデル比較バックテスト一式)。 再検証は py -X utf8 test/verify_course_features.py の1コマンド
  • 再検証の目安: コースデータがさらに2〜3ヶ月蓄積した時点(2026年9〜10月ごろ、展示タイム検証と同時期)。 評価期間を2期間以上に広げられるようになれば単一期間の偶然も潰せる
  • 全レースの予想明細: predictions_202605-06_course_features.csv / 集計値: verify_course_features_results.json

⚠ 単一期間(5〜6月)の検証。「有効なら即時導入」の判断基準に対して、 精度横ばい+回収率悪化のため導入見送りが妥当。features.py・本番モデルには一切手を加えていない。
