1. このシステムでできること

機械学習(LightGBM)が対象5場の全レースを毎朝予測し、「今日買うべきレースと買い目」だけを 1枚のレポートにまとめます。あなたがやることは、スマホでページを開いて、 「勝負」と書かれたレースをプラン通りに買うかどうか決めるだけです。

予測・レポート更新はすべてクラウド(GitHub)上で毎日自動実行されます。 自宅PCの電源が切れていても動きます。

2. スマホでの使い方

毎日の予想を見る

ブラウザ(Safari / Chrome)で次のURLを開きます。毎朝6時半ごろ自動で最新に変わります。

https://romancedawn59.github.io/boat-yosou/

ホーム画面に追加する(最初に1回だけ・推奨)

iPhone (Safari) の場合
  1. 上のURLをSafariで開きます。
  2. 画面下中央の共有ボタン(四角+上矢印)をタップします。
  3. メニューを下にスクロールし、「ホーム画面に追加」をタップします。
  4. 右上の「追加」をタップします。
Android (Chrome) の場合
  1. 上のURLをChromeで開きます。
  2. 右上の「︙」メニューをタップします。
  3. 「ホーム画面に追加」をタップします。
  4. 「追加」をタップします。

以後はホーム画面のアイコンをタップするだけで、アプリのように最新の予想が開きます。

予想を今すぐ更新したいとき(手動実行)

  1. ブラウザで github.com を開き、ログインします。
  2. リポジトリ romancedawn59/boat-yosou を開きます。
  3. Actions タブ →「予想レポート更新」→「Run workflow」→ 緑の「Run workflow」をタップします。
  4. 1〜2分待ってから予想ページを再読み込みします。

過去の予想を見る

URLの末尾に archive/日付_場名.html を付けます。
例: https://romancedawn59.github.io/boat-yosou/archive/2026-07-07_heiwajima.html
場名: heiwajima / edogawa / tokoname / amagasaki / wakamatsu

3. PCでの使い方

予想を見る

スマホと同じURLをブラウザで開くだけです。加えてPCでは以下の2通りの方法があります。

方法手順用途
クラウド版(推奨) https://romancedawn59.github.io/boat-yosou/ を開く 日常利用。常に最新
ローカル版 Y:\マイドライブ\boat\reports\site\index.html をダブルクリック PCの自動実行(毎朝6:30)の出力。クラウドが止まった時の予備

手動でデータ取得・予想を実行する

コマンドプロンプトを開き、次を実行します(通常は不要。自動実行が止まっていた時用)。

cd /d Y:\マイドライブ\boat\src
py collect.py            … 最新データを取得します(前回の続きから自動)
py predict.py today      … 今日の予想レポートを生成します
py predict.py 2026-07-11 … 日付を指定して予想します

メンテナンス(月1回、自動でも実行される)

py train_model.py        … モデルを再学習します
py backtest.py           … 買い方別の回収率を再検証します
再学習したら必ず py backtest.py を実行し、「荒れ注意のみ回収率100%超」の 前提が維持されているか確認します。崩れていたら買い目構成の見直しが必要です。

4. レポートのデータの見方

① ページ最上部: 本日のまとめ

黄色い枠に「本日の勝負レース: 平和島5R、尼崎2R…(予算 ◯円)」と表示されます。 まずここだけ見れば今日の行動が決まります。「本日は勝負レースなし」の日は買いません。

② レースカードの読み方

表示意味
締切 HH:MM投票締切の予定時刻です。この時刻までに投票します
本命勝負所買う対象のレース(荒れ注意=検証済みエッジ)。1日最大10レース
準勝負所本命が10未満の日の補充枠。期待値は本命より低い(参考)
堅め予測1位艇の勝率50%以上。順当決着が濃厚です
標準予測1位艇の勝率35〜50%です
荒れ注意予測1位艇の勝率35%未満。波乱含み=このシステムの狙い目です
予報: 風速◯m/s…レース時刻の気象予報(Open-Meteo)です。参考表示のみで予測には使っていません

③ 選手表の読み方

各レースの表はモデルの予測勝率が高い順に並んでいます(上の艇ほど本命評価です)。 左端の数字は枠番で、実際のボートカラー(1=白、2=黒、3=赤、4=青、5=黄、6=緑)で色分けされています。 「%」はその艇が1着になる予測確率です。

④ 買い目プランの読み方

表記券種読み方
1=2=43連複1・2・4号艇が順位を問わず1〜3着に入れば的中
2-1-33連単2号艇が1着、1号艇が2着、3号艇が3着、この順番どおりで的中
1=32連複1・3号艇が順位を問わず1〜2着なら的中(参考買い目で使用)

⑤ 3人の予想者と予想屋ken

各レースには3人の予想者の視点と、それをまとめた予想屋kenの購入プランが載ります:

予想者スタイル内容
A 石橋渡堅実(ローリスク)発生確率の高い2連複・3連複を5点。的中率約70%だが配当は安い
B 山田三連単のびのび発生確率上位の3連単を5点。的中率約33%
C 勝万舟一発狙い万舟圏(発生確率0.5%以下=実績平均払戻2.2万円)から発生率の高い順に5点
予想屋kenまとめ役水色枠・金額つき。実際に買うのはこれ。1レース1,000円で3人の案を配合(C案を必ず100円以上含む)。荒れ注意レースでは検証済み構成(3連複軸1流し+3連単穴)を核に維持

A/B/Cには購入額を付けません(視点の提示と通算成績の記録用)。それぞれの通算成績は ナビの「通算成績」ページで毎晩更新されます。推奨運用は「本命勝負所のkenプランだけ買う」です。

⑥ 「オッズ反映⏱」タブ(毎日12:00更新・参考)

12:00時点で締切前のレースには、カード内に「朝の予想 ⇔ オッズ反映⏱」のタブが付きます。 オッズ反映タブでは、kenプラン各点の現在オッズ・想定払戻・EV(モデル確率×オッズ)と、 実験枠の「オッズ妙味」上位3点が見られます。

  • 成績対象外の参考情報です(採点・通算成績・勝負所判定には一切影響しない)
  • 検証ではEVによる目の選別は逆効果だったため、判断材料の提示にとどめています
  • 朝の予想(タブ1)は終日変わりません。12:00より前の締切レースにはタブが付きません

5. 買い目の考え方と資金管理

1年分のデータで「学習に使っていない期間だけで成績を測る」検証(ウォークフォワード)を行った結果:

  • 「荒れ注意」レースだけが期待値プラス(全場合算で回収率122%、5期間すべて104%以上)
  • 堅め・標準のレースは、どの買い方でも回収率77〜90%で一貫してマイナス → 見送りが正解
  • 対象5場の直近3ヶ月テストでは回収率159%(3ヶ月すべて100%超、最大の1発を除いても109%)
大事な性質: 勝ち日は約36%で、負け日の方が多い戦略です。利益はときどき来る 高配当がまとめて運んできます。勝負レースを機械的に全部買い続けることが前提で、 調子で買ったり買わなかったりすると、肝心の高配当を逃して負け側に落ちます。
  • 1日の投資目安: 勝負レース数×1,000円(平均5レース前後=約5,000円)
  • 検証中の最大ドローダウン(一時的な最大の沈み)は約2.7万円。5万円程度の余裕資金で運用してください

6. 自動で動いている処理

クラウド(GitHub Actions) — メイン。PC電源に無関係

いつ処理
毎日 朝6:30ごろデータ収集 → 5場の予測 → 予想ページを自動更新 → LINEに通知(勝負レース・予算・レポートURL。bot「Ken」から届く)
毎日 12:00ごろ締切前レースの最新オッズを取得し、各レースに「オッズ反映⏱」タブを追加(参考・成績対象外)
毎晩 23:30ごろ当日の全予想(A/B/C/ken)を確定結果で採点し、通算成績ページを更新
毎月1日 朝5:00ごろ最新1年分でモデルを再学習し、バックテストで検証

自宅PC(タスクスケジューラ) — 予備系

いつ処理
毎日 6:00 / 6:30データ収集 / 予想レポートをGoogle Drive(reports/)に出力
毎日 9:00〜21:00各レース締切の15分前に1回だけ、展示タイムと直前オッズ(3連単/3連複全通り)のスナップショットを記録。保存のみで予測には未使用。数ヶ月蓄積後に朝予想との比較検証を行う
毎月1日 5:00再学習+バックテスト

PC側はスリープからは自動復帰して実行しますが、シャットダウン中は動きません (その間もクラウド側は動き続けます)。展示タイムの記録だけはPC側のみの機能です。

7. トラブルシューティング

⚠ よくある症状と対処

症状対処
予想ページが昨日のまま対象5場がすべて非開催の日は更新されません。開催日なのに古い場合は、 「2. スマホでの使い方」の手動実行を行ってください
ページが開かない(404等)GitHub側の一時障害の可能性があります。数分待って再読み込みしてください
気象予報が表示されない気象APIの一時的な接続失敗です。買い目には影響ありません
PC側のレポートが出ていないY:\マイドライブ\boat\logs\predict.log の末尾を確認してください
手動実行でエラーになるcd /d Y:\マイドライブ\boat && py -m pip install -r requirements.txt を実行してから再試行してください

8. データ源と注意事項

  • BoatraceOpenAPI(非公式・無料): 番組表・レース結果。約30分間隔で更新
  • Open-Meteo(無料): 各場の座標のピンポイント気象予報。表示専用
  • boatrace.jp 直前情報: 展示タイム等。PC側で記録のみ
  • いずれかが停止した場合は代替源の再調査が必要です(非公式サービスのため)

仕組み全体の図解は flow.html、検証の詳細は test/backtest_report_*.html を参照してください。

予測は過去データに基づく参考情報であり、的中や利益を保証するものではありません。 舟券の購入は自己責任で、余裕資金の範囲で行ってください。
